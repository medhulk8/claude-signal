Claude Signal — Privacy Policy

Last updated: March 2026

Claude Signal is a Chrome extension that surfaces high-signal updates from Claude Code and the Anthropic developer platform. This policy explains what data the extension accesses and how it is used.

Data we collect

We do not collect any personal data. The extension does not transmit any information about you, your browser, or your activity to any server we operate.

Data stored locally

The extension stores the following data in your browser's local storage (chrome.storage.local), which never leaves your device:

  • cachedDigest — a cached copy of the public update feed fetched from GitHub Pages
  • seenIds — a record of which items you have already read
  • notifiedIds — a record of which items have already triggered a notification, to prevent repeat alerts
  • lastFetchedAt — the timestamp of the last successful feed fetch

This data is stored solely to provide the extension's core functionality. It is not shared with or accessible by anyone other than you.

Network requests

The extension periodically fetches a publicly available JSON file from GitHub Pages (medhulk8.github.io/claude-signal/digest.json). This is a one-way read — no data about you is included in this request beyond standard HTTP headers sent by your browser.

Permissions used

  • storage — to save the digest cache and your read state locally in your browser
  • alarms — to schedule a background fetch of the update feed every 60 minutes
  • notifications — to show a browser notification when new updates are available

Third-party services

The update feed is hosted on GitHub Pages. Fetching it is subject to GitHub's privacy policy. No other third-party services are used by the extension.

Contact

Questions or concerns: open an issue at github.com/medhulk8/claude-signal.
